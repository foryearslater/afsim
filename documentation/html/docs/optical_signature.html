optical_signature¶

optical_signature¶
optical_signature <signature-name> (model-name)
   ... model specific commands ...
end_optical_signature

Overview¶

optical_signature defines the optical signature for a platform type. The optical signature is used when a sensor attempts to detect a platform and needs the projected area of the platform in the direction of the sensor.

<signature-name> is the name to be given to the signature. If the name specifies the name of an existing definition then the new definition will replace the current one (i.e. the last occurrence will be used for the simulation.)

(model-name) is an optional parameter that specifies the implementation model to be used for defining the signature. Available models are:

  • WSF_OPTICAL_SIGNATURE - Defines the signature using azimuth/elevation tables. This is the default model if one is not explicitly chosen.

  • WSF_SPACE_OPTICAL_SIGNATURE - Computes a dynamic signature for space platforms.

  • WSF_COMPOSITE_OPTICAL_SIGNATURE - A simple dynamic optical and infrared signature.
