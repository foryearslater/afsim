Warning

The following weapon commands are not compatible with WSF_LASER_WEAPON. Attempting to use these commands will result in a warning or error. Avoid using WsfWeapon script methods which access the parameters of these commands.

  • quantity

  • maximum_quantity

  • reload_increment

  • reload_inventory

  • reload_threshold

  • reload_time

  • inhibit_while_reloading

Overview¶

WSF_LASER_WEAPON represents a High-Energy Laser (HEL) weapon. Most of the inputs for this weapon are for the fluence_model, which determines the propagation of laser energy and deposition on target, and thermal-related inputs, which limit the time that the laser can fire before having to dissipate excess thermal energy.

Commands¶

firing_time <time-value>¶

Specify the firing time per shot.

Note

If no firing_time is set by the user, the laser will continue to fire until either the user terminates the engagement with a call to CeaseFire, or the thermal system prevents lasing.

firing_update_interval <time-value>¶

Specify the interval at which the engagement will be updated and fluence on target integrated.

Default: Either 0.1 * firing_time, if firing_time is set, or 0.1 second, if not.

Non-Thermal System Related Commands¶

These commands are valid if no :command:`thermal_system` is specified.

number_of_shots <integer-value>¶

Specify the number of shots the laser can fire before it must cool down for a specified interval (see cooling_time). Either these inputs are used, or they are determined automatically with a thermal_system.

Default: 1000000 (unlimited)

cooling_time <time-value>¶

Specify a cooldown time, starting after the specified number_of_shots has been exhausted. Once this time interval has elapsed, the original shot number will be restored.

Note

cooling_time and number_of_shots are used as a low fidelity alternative to a thermal_system. One can use one or the other with this weapon.

Default: 30 seconds

Thermal System Related Commands¶

These commands are valid if a :command:`thermal_system` is specified.

cooling_update_interval <time-value>¶

Specify an interval to check whether the cooling cycle is complete.

Default: 10.0 seconds

efficiency <real-value>¶

Specifies the efficiency of the HEL system for converting input power into output laser power.

Note

For diode-based systems this value is approximately 0.15 - 0.20).

Default: 0.3

high_temperature_limit <temperature-value>¶

This is the temperature at which firing ceases and cooldown begins (the thermal system does not accept heat above this temperature).

Default: 100 celsius

low_temperature_limit <temperature-value>¶

This is the temperature at which maximum firing time is available (the thermal system does not cool below this temperature).

Default: -20 celsius

minimum_total_firing_time <time-value>¶

When in a cooldown cycle, the weapon will not be available until this much firing time has been restored to the system.

Default: 5 seconds
